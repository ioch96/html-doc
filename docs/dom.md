# Τι είναι το DOM

Τα αρχικά **DOM** σημαίνουν:

> **Document Object Model**

Στα ελληνικά:

> **Μοντέλο Αντικειμένων Εγγράφου**

Το DOM είναι μια **αναπαράσταση της HTML σε μορφή δέντρου (tree structure)**.

Ο browser δεν αποθηκεύει απλώς το HTML ως κείμενο.

Αντίθετα, δημιουργεί μια οργανωμένη δομή από αντικείμενα που αντιστοιχούν στα HTML στοιχεία.

Με απλά λόγια:

> **Το DOM είναι ο τρόπος με τον οποίο ο browser οργανώνει την HTML στη μνήμη του.**

---

## Παράδειγμα

HTML:

```html
<html>

<body>

<h1>Η σελίδα μου</h1>

<p>Καλώς ήρθατε.</p>

</body>

</html>
```

Ο browser το μετατρέπει περίπου στην εξής δομή:

```text
html
└── body
    ├── h1
    │   └── "Η σελίδα μου"
    │
    └── p
        └── "Καλώς ήρθατε."
```

Παρατήρησε ότι κάθε στοιχείο βρίσκεται μέσα σε κάποιο άλλο.

---

## Γιατί το DOM μοιάζει με δέντρο

Επειδή τα HTML στοιχεία μπορούν να περιέχουν άλλα HTML στοιχεία.

Παράδειγμα:

```html
<body>

    <article>

        <h2>Τίτλος</h2>

        <p>Κείμενο</p>

    </article>

</body>
```

Η δομή γίνεται:

```text
body
└── article
    ├── h2
    └── p
```

Το `<article>` περιέχει δύο στοιχεία.

Το `<body>` περιέχει το `<article>`.

Έτσι δημιουργείται μια ιεραρχία, όπως ακριβώς τα κλαδιά ενός δέντρου.

---

## Γονείς και παιδιά (Parent - Child)

Στο DOM χρησιμοποιούνται συχνά οι όροι:

- Parent (γονέας)
- Child (παιδί)
- Sibling (αδελφό στοιχείο)

Παράδειγμα:

```html
<body>

    <h1>HTML</h1>

    <p>Παράγραφος</p>

</body>
```

DOM:

```text
body
├── h1
└── p
```

Εδώ:

- το `<body>` είναι ο **γονέας**,
- τα `<h1>` και `<p>` είναι **παιδιά**,
- τα `<h1>` και `<p>` είναι επίσης **αδελφά στοιχεία**, επειδή έχουν τον ίδιο γονέα.

---

## Κάθε HTML στοιχείο γίνεται ένας κόμβος (Node)

Στο DOM κάθε στοιχείο ονομάζεται **Node** (κόμβος).

Παράδειγμα:

```html
<p>Γεια σου!</p>
```

δημιουργεί:

```text
p
└── "Γεια σου!"
```

Το `<p>` είναι ένας κόμβος.

Το κείμενο **"Γεια σου!"** είναι επίσης ένας κόμβος (Text Node).

Άρα το DOM δεν αποτελείται μόνο από tags.

Περιλαμβάνει και το ίδιο το κείμενο.

---

## Rendering και DOM

Οι δύο έννοιες συνδέονται στενά αλλά **δεν είναι το ίδιο πράγμα**.

Το **Rendering** είναι η διαδικασία.

Το **DOM** είναι το αποτέλεσμα ενός σημαντικού μέρους αυτής της διαδικασίας.

Μπορούμε να το φανταστούμε ως εξής:

```text
HTML

↓

Ο browser διαβάζει τον κώδικα

↓

Δημιουργεί το DOM

↓

Εμφανίζει τη σελίδα

↓

Rendering ολοκληρώθηκε
```
