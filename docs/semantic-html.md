# Semantic HTML

Κατά τη δημιουργία ιστοσελίδων χρησιμοποιούμε έναν συνδυασμό από **μη σημασιολογικά (non-semantic)** και **σημασιολογικά (semantic)** στοιχεία HTML.

Η λέξη **semantic** σημαίνει **«σχετικός με τη σημασία»**. Επομένως, τα σημασιολογικά στοιχεία HTML παρέχουν πληροφορίες σχετικά με το **είδος του περιεχομένου** που βρίσκεται ανάμεσα στο tag ανοίγματος και στο tag κλεισίματος.

Χρησιμοποιώντας τη **Semantic HTML**, επιλέγουμε τα στοιχεία HTML με βάση **τη σημασία και τον ρόλο τους**, και όχι με βάση την εμφάνισή τους.

Για παράδειγμα, τα στοιχεία `<div>` και `<span>` **δεν είναι σημασιολογικά στοιχεία**, επειδή δεν παρέχουν καμία πληροφορία σχετικά με το περιεχόμενό τους.

Αντί να χρησιμοποιήσουμε ένα στοιχείο `<div>` για να περιέχει την κεφαλίδα της σελίδας, μπορούμε να χρησιμοποιήσουμε το στοιχείο `<header>`, το οποίο δηλώνει ότι το περιεχόμενο αποτελεί την κεφαλίδα μιας σελίδας ή μιας ενότητας.

Παράδειγμα:

## Μη σημασιολογική HTML

```html
<div>
  <h1>My Website</h1>
</div>
```

## Σημασιολογική HTML

```html
<header>
  <h1>My Website</h1>
</header>
```

Στη δεύτερη περίπτωση, ακόμη και χωρίς να δούμε το περιεχόμενο, γνωρίζουμε ότι το στοιχείο περιέχει την **κεφαλίδα** της σελίδας.

---

## Γιατί να χρησιμοποιούμε Semantic HTML

### 1. Προσβασιμότητα (Accessibility)

Η Semantic HTML κάνει τις ιστοσελίδες πιο προσβάσιμες για:

- Άτομα με αναπηρίες.
- Χρήστες που χρησιμοποιούν **screen readers**.
- Κινητές συσκευές.

Αυτό συμβαίνει επειδή οι browsers και οι τεχνολογίες υποβοήθησης μπορούν να κατανοήσουν καλύτερα τη δομή της ιστοσελίδας.

---

### 2. Βελτιστοποίηση για Μηχανές Αναζήτησης (SEO)

Η Semantic HTML συμβάλλει στη βελτίωση του **SEO (Search Engine Optimization)**.

Το SEO είναι η διαδικασία βελτίωσης της κατάταξης μιας ιστοσελίδας στα αποτελέσματα των μηχανών αναζήτησης, όπως η Google.

Χρησιμοποιώντας σημασιολογικά στοιχεία, οι μηχανές αναζήτησης μπορούν να αναγνωρίσουν πιο εύκολα:

- Τον κύριο τίτλο.
- Το κύριο περιεχόμενο.
- Την πλοήγηση.
- Το υποσέλιδο.
- Τις επιμέρους ενότητες της σελίδας.

Έτσι κατανοούν καλύτερα το περιεχόμενο της ιστοσελίδας.

---

### 3. Πιο κατανοητός κώδικας

Η Semantic HTML κάνει τον πηγαίο κώδικα πολύ πιο ευανάγνωστο.

Παράδειγμα:

```html
<div id="header">
  ...
</div>

<div id="navigation">
  ...
</div>

<div id="content">
  ...
</div>

<div id="footer">
  ...
</div>
```

είναι λιγότερο κατανοητό από:

```html
<header>
  ...
</header>

<nav>
  ...
</nav>

<main>
  ...
</main>

<footer>
  ...
</footer>
```

Ακόμη και χωρίς να διαβάσουμε το περιεχόμενο, μπορούμε να καταλάβουμε αμέσως τη δομή της ιστοσελίδας.

---

## Semantic vs Non-semantic HTML

### Non-semantic στοιχεία

Τα παρακάτω στοιχεία **δεν περιγράφουν** το περιεχόμενό τους:

- `<div>`
- `<span>`

Αποτελούν γενικά containers.

---

### Semantic στοιχεία

Τα παρακάτω στοιχεία περιγράφουν τον σκοπό του περιεχομένου τους:

- `<header>`
- `<nav>`
- `<main>`
- `<section>`
- `<article>`
- `<aside>`
- `<footer>`
- `<figure>`
- `<figcaption>`

---

## Ένα απλό παράδειγμα

### Χωρίς Semantic HTML

```html
<div>
  <div>
    <h1>My Blog</h1>
  </div>

  <div>
    <div>
      <h2>HTML</h2>
      <p>...</p>
    </div>
  </div>

  <div>
    Copyright 2026
  </div>
</div>
```

Δεν είναι εύκολο να καταλάβουμε ποιο `<div>` είναι η κεφαλίδα, το κύριο περιεχόμενο ή το υποσέλιδο.

---

### Με Semantic HTML

```html
<header>
  <h1>My Blog</h1>
</header>

<main>

  <article>
    <h2>HTML</h2>
    <p>...</p>
  </article>

</main>

<footer>
  Copyright 2026
</footer>
```

Η δομή της σελίδας είναι πλέον ξεκάθαρη.
