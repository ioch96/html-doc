# DOM (Document Object Model)

Μετά την ολοκλήρωση του HTML Parsing, ο browser έχει πλέον αναλύσει ολόκληρο το HTML έγγραφο.

Ωστόσο, ο HTML κώδικας από μόνος του δεν είναι αρκετός για να μπορεί ο browser να διαχειρίζεται εύκολα τα στοιχεία της ιστοσελίδας.

Για τον λόγο αυτό δημιουργεί μια εσωτερική αναπαράσταση του εγγράφου, γνωστή ως **DOM (Document Object Model)**.

Το DOM είναι μία από τις σημαντικότερες έννοιες στην ανάπτυξη ιστοσελίδων, καθώς αποτελεί τη δομή πάνω στην οποία βασίζονται ο browser και η JavaScript.

---

## Τι είναι το DOM;

Το **Document Object Model (DOM)** είναι η εσωτερική αναπαράσταση ενός HTML εγγράφου στη μνήμη του browser.

Το HTML έγγραφο μετατρέπεται σε μια δενδρική δομή (Tree Structure), όπου κάθε HTML στοιχείο αντιστοιχεί σε έναν κόμβο (Node).

Με αυτόν τον τρόπο ο browser μπορεί να γνωρίζει:

* ποια στοιχεία υπάρχουν,
* ποια στοιχεία περιέχουν άλλα στοιχεία,
* ποια είναι η μεταξύ τους σχέση,
* και σε ποια θέση βρίσκεται κάθε στοιχείο μέσα στο έγγραφο.

---

## Γιατί δημιουργείται το DOM;

Ο browser δημιουργεί το DOM ώστε να μπορεί να διαχειρίζεται αποτελεσματικά τη δομή της ιστοσελίδας.

Χάρη στο DOM μπορεί:

* να εμφανίσει σωστά την ιστοσελίδα,
* να εφαρμόσει τους κανόνες CSS,
* να ενημερώνει τη διάταξη όταν αλλάζει το περιεχόμενο,
* να επιτρέπει στη JavaScript να αλληλεπιδρά με τα στοιχεία της σελίδας.

Χωρίς το DOM, ο browser θα έπρεπε να διαβάζει ολόκληρο το HTML έγγραφο από την αρχή κάθε φορά που χρειαζόταν να εντοπίσει ένα στοιχείο.

---

## Το DOM είναι μια δενδρική δομή

Το DOM οργανώνει όλα τα στοιχεία του HTML εγγράφου σε μορφή δέντρου.

Για παράδειγμα, το παρακάτω HTML έγγραφο:

```html id="k2v7fs"
<!DOCTYPE html>
<html>
<head>
    <title>Παράδειγμα</title>
</head>

<body>
    <h1>Hello</h1>
    <p>World</p>
</body>
</html>
```

μετατρέπεται σε μια ιεραρχική δομή.

```text id="tmng8w"
Document
│
└── html
    ├── head
    │   └── title
    │       └── "Παράδειγμα"
    │
    └── body
        ├── h1
        │   └── "Hello"
        │
        └── p
            └── "World"
```

Κάθε στοιχείο αποτελεί έναν κόμβο του δέντρου.

---

## Γονείς και παιδιά

Επειδή το DOM έχει μορφή δέντρου, κάθε κόμβος μπορεί να έχει σχέσεις με άλλους κόμβους.

Οι πιο συνηθισμένες σχέσεις είναι:

* **Parent (Γονέας)**, το στοιχείο που περιέχει ένα άλλο στοιχείο.
* **Child (Παιδί)**, το στοιχείο που βρίσκεται μέσα σε ένα άλλο στοιχείο.
* **Sibling (Αδελφό στοιχείο)**, στοιχεία που έχουν τον ίδιο γονέα.

Για παράδειγμα:

```html id="4s2rq7"
<body>
    <h1>Hello</h1>
    <p>World</p>
</body>
```

Στο παραπάνω παράδειγμα:

* το `<body>` είναι ο γονέας των `<h1>` και `<p>`,
* τα `<h1>` και `<p>` είναι παιδιά του `<body>`,
* τα `<h1>` και `<p>` είναι επίσης αδέλφια (siblings), επειδή έχουν τον ίδιο γονέα.

Η κατανόηση αυτών των σχέσεων είναι σημαντική, καθώς χρησιμοποιούνται τόσο από τον browser όσο και από τη JavaScript.

---

## DOM και HTML

Το DOM δημιουργείται από το HTML έγγραφο, αλλά δεν είναι το ίδιο το HTML.

Το HTML είναι ένα αρχείο κειμένου που περιέχει tags και περιεχόμενο.

Το DOM είναι μια δυναμική δομή δεδομένων που δημιουργείται στη μνήμη του browser κατά τη διάρκεια του Rendering.

Με άλλα λόγια:

* το HTML περιγράφει τη δομή της ιστοσελίδας,
* το DOM είναι η αναπαράσταση αυτής της δομής στη μνήμη.

---

## Το DOM χρησιμοποιείται από τον browser

Το DOM δεν δημιουργείται αποκλειστικά για τη JavaScript.

Ο ίδιος ο browser το χρησιμοποιεί για:

* την εφαρμογή των κανόνων CSS,
* τον υπολογισμό της διάταξης της σελίδας,
* την ενημέρωση της οθόνης όταν αλλάζει το περιεχόμενο,
* τη διαχείριση γεγονότων (events),
* την επανασχεδίαση (repaint) και την αναδιάταξη (reflow) της ιστοσελίδας.

Η JavaScript αποτελεί απλώς μία από τις τεχνολογίες που μπορούν να αλληλεπιδράσουν με το DOM.

---

## Το επόμενο στάδιο

Αφού δημιουργηθεί το DOM, ο browser συνεχίζει τη διαδικασία του Rendering αναλύοντας τους κανόνες CSS.

Το αποτέλεσμα αυτής της διαδικασίας είναι η δημιουργία του **CSSOM (CSS Object Model)**, το οποίο θα εξετάσουμε στο επόμενο κεφάλαιο.

---

## Σύνοψη

Το DOM είναι η εσωτερική αναπαράσταση ενός HTML εγγράφου στη μνήμη του browser.

Κατά τη δημιουργία του:

* κάθε HTML στοιχείο μετατρέπεται σε έναν κόμβο,
* οι κόμβοι οργανώνονται σε μορφή δέντρου,
* αποτυπώνονται οι σχέσεις γονέα, παιδιού και αδελφών στοιχείων,
* ο browser αποκτά μια δομή που μπορεί να χρησιμοποιήσει για το Rendering και για την αλληλεπίδραση με τη σελίδα.

Το DOM αποτελεί τη γέφυρα ανάμεσα στον HTML κώδικα και στη λειτουργία του browser.
