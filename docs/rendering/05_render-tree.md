# Render Tree

Μέχρι αυτό το σημείο ο browser έχει δημιουργήσει δύο σημαντικές δομές:

* το **DOM (Document Object Model)**, που περιγράφει τη δομή του HTML εγγράφου,
* το **CSSOM (CSS Object Model)**, που περιγράφει όλους τους κανόνες μορφοποίησης.

Ωστόσο, καμία από αυτές τις δύο δομές δεν είναι αρκετή από μόνη της για να εμφανιστεί η ιστοσελίδα στην οθόνη.

Για τον λόγο αυτό ο browser δημιουργεί μια νέα δομή, γνωστή ως **Render Tree**.

---

## Τι είναι το Render Tree;

Το **Render Tree** είναι μια εσωτερική δομή που δημιουργείται από τον browser συνδυάζοντας το **DOM** και το **CSSOM**.

Κάθε κόμβος του Render Tree αντιπροσωπεύει ένα στοιχείο που πρόκειται να εμφανιστεί στην οθόνη μαζί με τις ιδιότητες μορφοποίησής του.

Με άλλα λόγια:

* το DOM περιγράφει **ποια στοιχεία υπάρχουν**,
* το CSSOM περιγράφει **πώς πρέπει να μορφοποιηθούν**,
* το Render Tree περιγράφει **τι θα εμφανιστεί τελικά στην οθόνη**.

---

## Πώς δημιουργείται το Render Tree;

Ο browser εξετάζει κάθε στοιχείο του DOM.

Για κάθε στοιχείο:

1. αναζητά τους κανόνες CSS που το αφορούν,
2. υπολογίζει ποιες ιδιότητες ισχύουν,
3. δημιουργεί έναν νέο κόμβο στο Render Tree.

Η διαδικασία επαναλαμβάνεται μέχρι να επεξεργαστεί όλα τα στοιχεία που πρέπει να εμφανιστούν.

---

## Παράδειγμα

Ας θεωρήσουμε το παρακάτω HTML:

```html
<body>
    <h1>Hello</h1>
    <p>World</p>
</body>
```

και το αντίστοιχο CSS:

```css
h1 {
    color: blue;
}

p {
    color: gray;
}
```

Το DOM περιέχει τη δομή των στοιχείων:

```text
body
├── h1
└── p
```

Το CSSOM περιέχει τους κανόνες μορφοποίησης:

```text
h1 → color: blue

p → color: gray
```

Ο browser συνδυάζει αυτές τις δύο πληροφορίες και δημιουργεί το Render Tree:

```text
body
├── h1 (color: blue)
└── p (color: gray)
```

Παρατηρούμε ότι πλέον κάθε κόμβος περιέχει όχι μόνο το HTML στοιχείο αλλά και τις ιδιότητες που θα χρησιμοποιηθούν για την εμφάνισή του.

---

## Δεν συμμετέχουν όλα τα στοιχεία

Το Render Tree δεν περιέχει απαραίτητα όλα τα στοιχεία του DOM.

Για παράδειγμα, το στοιχείο `<head>` δεν εμφανίζεται ποτέ στην ιστοσελίδα.

Επομένως δεν συμμετέχει στο Render Tree.

Παράδειγμα:

```html
<html>
<head>
    <title>Example</title>
</head>

<body>
    <h1>Hello</h1>
</body>
</html>
```

Το DOM περιέχει τόσο το `<head>` όσο και το `<body>`.

Το Render Tree όμως περιέχει μόνο τα στοιχεία που πρόκειται να εμφανιστούν.

---

## Στοιχεία με `display: none`

Το ίδιο συμβαίνει και με στοιχεία που δεν εμφανίζονται λόγω CSS.

Παράδειγμα:

```html
<p class="hidden">Secret</p>
<p>Visible</p>
```

```css
.hidden {
    display: none;
}
```

Το στοιχείο με `display: none` υπάρχει κανονικά στο DOM.

Ωστόσο, επειδή δεν πρόκειται να εμφανιστεί στην οθόνη, δεν προστίθεται στο Render Tree.

Έτσι, το Render Tree περιέχει μόνο το δεύτερο στοιχείο.

---

## Γιατί δημιουργείται το Render Tree;

Το Render Tree περιέχει ακριβώς τις πληροφορίες που χρειάζεται ο browser για να εμφανίσει την ιστοσελίδα.

Για κάθε στοιχείο γνωρίζει:

* ότι πρέπει να εμφανιστεί,
* ποιες ιδιότητες CSS ισχύουν,
* ποια είναι η σχέση του με τα υπόλοιπα στοιχεία.

Ακόμη όμως δεν γνωρίζει την ακριβή θέση και το μέγεθός του μέσα στη σελίδα.

Αυτό υπολογίζεται στο επόμενο στάδιο.

---

## Το επόμενο στάδιο

Αφού ολοκληρωθεί η δημιουργία του Render Tree, ο browser πρέπει να υπολογίσει:

* τη θέση κάθε στοιχείου,
* το πλάτος και το ύψος του,
* τις αποστάσεις μεταξύ των στοιχείων,
* τη συνολική διάταξη της ιστοσελίδας.

Η διαδικασία αυτή ονομάζεται **Layout** (ή **Reflow**) και αποτελεί το επόμενο στάδιο του Rendering.

---

## Σύνοψη

Το Render Tree δημιουργείται από τον συνδυασμό του DOM και του CSSOM.

Κατά τη δημιουργία του:

* κάθε εμφανίσιμο HTML στοιχείο αντιστοιχίζεται στους κανόνες CSS που το αφορούν,
* εξαιρούνται στοιχεία που δεν εμφανίζονται, όπως το `<head>` ή στοιχεία με `display: none`,
* δημιουργείται μια νέα δομή που περιέχει μόνο όσα θα σχεδιαστούν στην οθόνη.

Το Render Tree αποτελεί τη βάση πάνω στην οποία ο browser θα υπολογίσει τη διάταξη και στη συνέχεια θα σχεδιάσει την ιστοσελίδα.
